# Harness Spec

## Design Principles

- Schema first: agents produce structured artifacts before notes.
- Evidence first: claims should link to source paths and short excerpts.
- Role bounded: each agent has a narrow responsibility.
- Replayable: every run creates durable artifacts under `runs/`.
- Provider neutral: model providers are adapters, not core logic.
- Human editable: JSON artifacts and Obsidian notes can be reviewed and corrected manually.

## State Machine

```text
CREATED
  -> SOURCE_FETCHED
  -> INVENTORY_BUILT
  -> STRUCTURE_ANALYZED
  -> WORKFLOW_ANALYZED
  -> PATTERNS_MINED
  -> ASSETS_RENDERED
  -> TRANSLATED
  -> REVIEWED
  -> PUBLISHED | NEEDS_REVISION | FAILED
```

## Artifact Directory

```text
runs/<run-id>/
  run.json
  sources.lock.json
  inventory.json
  skills/<skill-id>/
    source_snapshot.json
    structure_analysis.json
    workflow_analysis.json
    skill_note.md
    review_report.json
  patterns/patterns.json
  render_manifest.json
  errors.json
```

## Quality Gates

- Every skill must have source repo, commit, root path, and `SKILL.md` path.
- Every workflow step should contain evidence; unsupported inferred steps must be marked.
- Patterns from one example are candidates, not established patterns.
- Final notes must include frontmatter, source references, confidence, and evidence sections.
- Reviewer can block publishing when evidence is weak or claims are unsupported.

## Resume Behavior

- `asa resume --run runs/<run-id>` loads the original config path from `run.json` when available.
- Existing artifacts are reused only if they validate against the current schema.
- Missing or invalid artifacts are regenerated by the relevant agent stage.
- Final schema failures write a sibling `*.error.json` artifact and remain recoverable.

## GitHub Acquisition

- The collector first attempts shallow `git clone --filter blob:none`.
- On clone failure, it falls back to GitHub zip archives.
- Inventory records `source.acquisition_method` for auditability.

## Skill Limit

- `asa run --limit-skills N` limits analysis to the first N discovered skill packages.
- The aggregate inventory records only the selected packages and includes a warning when limiting occurred.
- This is intended for real-model smoke runs and cost control.

## Run Planning

- `asa plan-run` runs collectors only.
- It reports discovered skills, selected skills, provider/model, and estimated agent calls.
- It is safe to run before real-model analysis to estimate scope and cost.

## Review Summary

- `asa review-run --run runs/<run-id>` aggregates all `skills/*/review_report.json` files.
- It reports status counts, average quality scores, issue totals, unsupported claims, missing evidence, over-inference, and publish approval state.
- Use it after real-model runs to quickly decide whether artifacts are ready for review or need prompt/schema iteration.
